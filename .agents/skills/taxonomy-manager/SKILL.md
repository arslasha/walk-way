# Taxonomy Manager (Управляющий Таксономией Вайбов)

Этот скилл помогает поддерживать список эмоциональных тегов (вайбов) в актуальном и чистом состоянии, не давая ему разрастись до сотен дублирующихся категорий.

## Задачи скилла

1. **Аудит текущей таксономии:** Проверка списка вайбов (например, в `LLM_STRATEGY.md`) на предмет логических пересечений.
2. **Интеграция новых категорий:** Если появляется новый запрос от пользователей (например, "хочу места для работы с ноутбуком" или "где посидеть с собакой"), скилл должен предложить, как органично вписать это в таксономию.
3. **Консолидация:** Если база данных накопила мусорные теги (синонимы), скилл должен составить карту маппинга (например, объединить `Тихое место`, `Спокойно` и `Для интровертов` в один каноничный тег).

## Правила добавления новых вайбов
При предложении нового вайба всегда оценивайте его по трем критериям:
1. **Эмоциональность/Контекстность:** Это реально вайб или просто факт? ("Кофе с собой" — это факт/удобство. "Ламповая кофейня" — это вайб).
2. **Частотность:** Достаточно ли в базе мест (парков, баров, кафе), которые подойдут под этот тег?
3. **Отличие от существующих:** Не перекрывает ли он на 90% уже существующий тег?

## Как работать с запросами
- Если пользователь просит добавить новую категорию, проанализируйте ее по правилам выше.
- Предложите название категории на двух языках (Русский / Английский) для удобства кода.
- Если вы утверждаете новые теги, напомните пользователю (или обновите самостоятельно) файл `docs/LLM_STRATEGY.md` в блоке "Таксономия Вайбов", чтобы системный промпт LLM учитывал эти изменения в будущем.
