# Directive Writer (Генератор Директив для Layer 1)

В архитектуре Walk-Way (согласно `GEMINI.md`), **Layer 1: Directive** отвечает за хранение строгих инструкций (SOPs) для других агентов. Этот скилл помогает превращать хаотичные процессы в четкие алгоритмы.

## Правила написания Директив

Директива должна быть детерминированной, строгой и не допускать двойных толкований.

### Формат документа (Markdown)

1. **Trigger (Условие срабатывания):** 
   Четко опишите, когда агент должен использовать эту директиву.
   *Пример: "Используйте эту директиву каждый раз, когда необходимо добавить новое поле в модель Django."*

2. **Core Constraints (Ключевые ограничения):**
   Список "Что НЕЛЬЗЯ делать ни при каких обстоятельствах".
   *Пример: "ЗАПРЕЩАЕТСЯ применять миграции без создания файла миграции внутри Docker-контейнера."*

3. **Step-by-Step Workflow (Пошаговый процесс):**
   Разбейте задачу на логические шаги (Шаг 1, Шаг 2, ...).
   Указывайте конкретные консольные команды или пути к файлам, где это возможно.
   *Пример: "Шаг 1. Откройте `models.py`. Шаг 2. Добавьте поле. Шаг 3. Выполните `python manage.py makemigrations`."*

4. **Expected Output (Ожидаемый результат):**
   Что должно получиться в итоге? Какие файлы изменятся?
   
5. **Troubleshooting / Edge Cases:**
   Как решать типовые ошибки (например, конфликты слияния или ошибки БД).

## Процесс работы
1. Узнайте у пользователя, какой процесс нужно стандартизировать.
2. Напишите черновик Директивы. Обязательно добавьте блок `Trigger`.
3. Сохраните утвержденную директиву в корневую папку или специализированную папку для правил, например `directives/название-директивы.md`.
4. Если директива очень важная, предложите пользователю добавить ссылку на нее в `GEMINI.md` или `.clinerules`.
