# Стратегия LLM: Генерация Вайбов и Айсбрейкеров (Walk-Way)

## 1. Роль и Цели
Этот документ описывает стратегию использования LLM для обогащения сырых данных из KudaGo. Сырые описания мест часто сухие или сугубо информативные. Наша цель — вдохнуть в них жизнь, выделив эмоциональную окраску (вайбы) и создав контекстные темы для разговора (айсбрейкеры).

**Основные задачи LLM:**
1. **Классификация вайбов:** Присвоение каждому месту 1-3 эмоциональных тегов (`is_vibe=True`).
2. **Генерация айсбрейкеров:** Создание 1-2 неформальных, цепляющих тем для разговора или идей для взаимодействия в этом месте, идеально подходящих для свиданий или встреч.

## 2. Таксономия Вайбов (Vibe Categories)
Чтобы база тегов не превратилась в хаос, LLM должна опираться на заранее определенные категории или строгие гайдлайны. Вот предлагаемый список основных вайбов для первой версии:

### Атмосфера и Настроение (Atmosphere)
* `Романтично` (Romantic) — приглушенный свет, красивые виды, уединение.
* `Уютно` (Cozy) — мягкие кресла, тепло, выпечка, теплая атмосфера.
* `Эстетично` (Aesthetic) — фотогенично, красивый интерьер/архитектура, визуал.
* `Мрачно / Меланхолично` (Melancholic/Gothic) — для глубоких мыслей, дождливой погоды, индустриальных пейзажей.
* `Шумно / Активно` (Loud/Active) — музыка, толпа, энергия.

### Социальный Контекст (Social)
* `Для интровертов` (For Introverts) — тихо, можно помолчать, мало людей.
* `Идеально для первого свидания` (First Date) — безопасно, есть о чем поговорить, можно легко уйти.
* `Для глубоких бесед` (Deep Talks) — нет громкой музыки, располагает к откровениям.
* `Для больших компаний` (Group Hangout) — просторно, весело.

### Активность (Activity)
* `Подумать` (Intellectual) — книжные, лектории, тихие аллеи.
* `Расслабиться` (Relax) — пуфы, парки у воды, неспешно, спокойный отдых.
* `Приключения` (Adventurous) — заброшенные места, квесты, необычные маршруты, крыши.

*Примечание:* В будущем добавим привязку ко времени суток (например, "Вайб заката", "Ночной перекус").

## 3. Правила для Айсбрейкеров
Айсбрейкер не должен звучать как совет от Википедии или ИИ. Он должен быть живым, немного игривым и контекстным.
* **Плохо:** "Обсудите историю постройки этого здания в 18 веке."
* **Хорошо:** "Спорим, ты не угадаешь, что раньше было в этом здании? Выглядит как старый завод, а на деле..."
* **Плохо:** "Спроси, любит ли он/она кофе."
* **Хорошо:** "Возьмите два разных необычных рафа и устройте слепую дегустацию — кто угадает больше ингредиентов?"

## 4. Системный Промпт (System Prompt)

Ниже представлен черновик системного промпта для скрипта Layer 3:

```text
Ты — харизматичный местный гид и эксперт по свиданиям и прогулкам. Твоя задача — анализировать описания городских локаций (парки, кафе, музеи, выставки) и определять их эмоциональный "вайб", а также придумывать интересные "айсбрейкеры" (темы или действия для свидания в этом месте).

ПРАВИЛА ДЛЯ ВАЙБОВ:
Выбери от 1 до 3 наиболее подходящих тегов из списка:
[Романтично, Уютно, Эстетично, Мрачно, Шумно, Для интровертов, Для первого свидания, Для глубоких бесед, Подумать, Расслабиться, Приключения].
Если место категорически не подходит ни под один, можешь предложить свой ОДИН короткий тег, но старайся держаться списка.

ПРАВИЛА ДЛЯ АЙСБРЕЙКЕРОВ:
Придумай 1-2 идеи для взаимодействия на свидании в этом месте. Это может быть тема для обсуждения, вдохновленная локацией, или мини-игра.
Тон: Неформальный, легкий, флиртующий или интригующий. Никакой духоты и энциклопедичности. Пиши от второго лица (обращайся к пользователю). Максимум 2 предложения на один айсбрейкер.

ФОРМАТ ВЫВОДА:
Ты должен вернуть строго валидный JSON без маркдаун-оберток.
{
  "vibes": ["тег1", "тег2"],
  "icebreakers": [
    "Текст айсбрейкера 1",
    "Текст айсбрейкера 2"
  ]
}

ВХОДНЫЕ ДАННЫЕ:
Название: {place_name}
Категория: {place_category}
Описание: {place_description}
```

## 5. Интеграция с Pipeline (Layer 3)
* Скрипт (например, `backend/places/management/commands/generate_vibes.py`) будет брать объекты `Place`, у которых `is_analyzed=False` (или где нет вайб-тегов).
* Отправлять данные в LLM API с этим системным промптом.
* Парсить JSON.
* Создавать/находить объекты `Tag` с `is_vibe=True` и связывать их с местом.
* Сохранять айсбрейкеры в новую модель или JSONField в модели `Place` (например, `Place.icebreakers`).
