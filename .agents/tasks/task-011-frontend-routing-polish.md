# Task 011: Frontend Routing Polish & Loop Routing (Круговые маршруты)

**Assignee:** Frontend Agent
**Status:** Not Started

## Context
Согласно плану разработки, базовая интеграция построения маршрута (Task 010) выполнена, но фронтенд-часть требует доработки для полноценного запуска. Нам необходимо реализовать алгоритм круговых маршрутов (Loop Routing), чтобы пользователь мог выбрать желаемую продолжительность прогулки, а система построила замкнутый маршрут (A -> B -> C -> A), возвращающий его в исходную точку. 

Также необходимо отполировать интерфейс карты и флоу добавления мест.

## Requirements

1. **Интерфейс настройки маршрута (Route Builder UI)**
   - Добавить в `RouteBottomSheet` (или отдельный компонент) переключатель типа маршрута: "Обычный" (A -> B) или "Круговой" (Loop).
   - Для кругового маршрута добавить выбор желаемого времени прогулки (например, 1 час, 2 часа).
   - Обновить UI карточек "По пути" (Smart POIs), сделав их более информативными.

2. **Логика кругового маршрута (Frontend/Zustand)**
   - Расширить `routeStore.ts` для поддержки режима `isLoopRoute`.
   - Разработать алгоритм (или подготовить интеграцию с бэкендом), который при выборе одной точки и кругового режима будет автоматически подбирать 2-3 интересные точки поблизости (с учетом радиуса, зависящего от времени прогулки) и замыкать маршрут на исходной точке.
   - Отрисовка замкнутого полигона/линии на карте.

3. **UX Полировка Карты**
   - Улучшить переходы камеры (`flyTo` и `fitBounds`) при добавлении новых точек в маршрут, чтобы они не дергались.
   - Сделать так, чтобы при активном маршруте карта плавно зумировалась на весь маршрут целиком.
   - Добавить кнопку "Сбросить маршрут" на видное место.

## Constraints & Rules
- Все изменения в UI должны строго соответствовать `DESIGN.md`.
- Сохраняйте плавность анимаций, используйте `framer-motion` или классы `tw-animate-css` при необходимости.
- Следуйте протоколу `version-control-protocol.md`.
