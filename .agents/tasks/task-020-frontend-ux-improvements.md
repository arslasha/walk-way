# Task 020: Frontend UX/UI Improvements & Bug Fixes

**Assignee:** Frontend Agent
**Status:** To Do

## Context
На основе отзывов пользователей и тестирования интерфейса необходимо провести серию точечных доработок UI/UX, улучшить навигацию, исправить проблемы со скроллом тегов, добавить модальное окно деталей места и сделать уведомления более интерактивными.
- **References:** `PLAN.md` (Phase 2 / Phase 3 Polish), `DESIGN.md`.

## Requirements

### 1. Map Tooltips (Тултипы на карте)
- **Текущее состояние:** Показывают только название места.
- **Требование:** Отображать в тултипе (Popup/Tooltip) уменьшенную фотографию места (аватар 48x48px или 64x64px со скругленными углами) и мини-описание (ровно одно предложение, обрезать с помощью `line-clamp-1` или `line-clamp-2` при необходимости).
- **Стилизация:** Тултип должен оставаться компактным и не перегружать карту. Использовать Flexbox layout (фотография слева, название и описание справа). Максимальная ширина всплывающего окна — 240-280px.

### 2. Header & Navigation Clean-up
- **Требование:** 
  - Удалить кнопку "Зарегистрироваться" (Sign Up) из верхней панели навигации (Header). Переход на регистрацию должен осуществляться исключительно со страницы входа (Sign In), как это реализовано сейчас.
  - Удалить кнопку "Сохраненные" (Saved) из панели навигации. Переход в сохраненные папки и места должен происходить через личный кабинет (профиль) и кнопку справа, как реализовано сейчас.

### 3. Unified Preloader Component (Единый прелоадер)
- **Требование:** Создать или выделить в отдельный переиспользуемый компонент единый прелоадер (Loader), стилизованный аналогично тому, который используется на карте.
- **Гибкость:** Компонент должен принимать строковый параметр `description` или `message` для отображения кастомного текста загрузки под спиннером/анимацией.
- **Интеграция:** Использовать этот прелоадер на странице с карточками мест в момент, когда загружается раздел с местами рядом с уже выбранной пользователем точкой.

### 4. Favorite Active Status on Place Cards
- **Требование:** Иконка избранного (сердечко/закладка) на карточках мест (`PlaceCard`) должна подсвечиваться как активная (закрашенная/активный цвет) в том случае, если это место добавлено пользователем в *любую* из его папок (коллекций) или общий список избранного.
- **Реактивность:** Состояние должно мгновенно обновляться в UI при добавлении или удалении места из любой коллекции пользователя.

### 5. Place Detail Modal (Модалка подробностей)
- **Требование:** При клике на карточку места (как в Explore Feed, так и в списке шторки) должно открываться элегантное модальное окно (Modal/Dialog) с подробной информацией.
- **Содержимое модального окна:**
  - Крупное качественное фото места.
  - Название места, категория (с иконкой) и полный текст описания.
  - Точный адрес и гео-координаты.
  - Список всех тегов (вайбов) места.
  - Кнопки быстрых действий (добавить в активный маршрут, сохранить в коллекцию).
  - Удобная кнопка закрытия (крестик в углу + закрытие по клавише Esc и клику на оверлей).

### 6. Clear All Button in Route Drawer (Кнопка очистки маршрута)
- **Требование:** В шторке с выбранными местами на странице карточек добавить кнопку "Очистить выбранное" (или "Очистить маршрут") с иконкой мусорной корзины.
- **Действие:** При клике на кнопку все выбранные места должны быть удалены из текущего активного списка/маршрута (полная очистка соответствующего Zustand-стора).

### 7. Toast Notifications Polish (Улучшение уведомлений)
- **Текущее состояние:** Уведомления появляются внизу справа, но их сложно быстро закрыть вручную.
- **Требование:** 
  - Добавить интерактивную кнопку-крестик (X) в правый верхний угол каждого тоста для мгновенного закрытия.
  - Реализовать скрытие уведомления при клике в любую пустую (свободную от других интерактивных элементов) область экрана (outside/backdrop click), либо кликом по самому телу тоста.

### 8. Horizontal Tag Scroll Fix (Исправление скролла тегов)
- **Проблема:** Меню выбора тегов (как на странице карточек мест, так и в панели фильтров на карте) не скроллится по горизонтали на некоторых разрешениях экрана.
- **Требование:** Обеспечить плавную горизонтальную прокрутку (Horizontal Scroll) для списка тегов. 
- **Реализация:** Использовать Tailwind-классы `flex overflow-x-auto whitespace-nowrap scrollbar-none` (или аналог для скрытия нативного скроллбара, сохраняя функционал прокрутки свайпом/колесиком мыши). Добавить визуальный градиент по краям, показывающий наличие скрытых элементов.
