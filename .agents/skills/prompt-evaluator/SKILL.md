# Prompt Evaluator (Оценщик и оптимизатор промптов)

Этот скилл предназначен для систематического тестирования и улучшения системных промптов (например, для генерации вайбов и айсбрейкеров). Он помогает убедиться, что LLM стабильно возвращает нужный формат и качественный контент на разных входных данных.

## 1. Подготовка тестовой выборки (Test Set)
Для начала работы попросите пользователя предоставить (или соберите сами из базы/API) 3-5 разнообразных примеров входных данных.
Например, для `LLM_STRATEGY.md` это могут быть сырые описания мест из KudaGo (парк, шумный бар, музей, необычное кафе).

## 2. Критерии оценки (Evaluation Criteria)
Перед запуском промпта определите с пользователем, по каким критериям мы оцениваем результат. Стандартные критерии:
- **Точность формата:** Вернулся ли строго валидный JSON без маркдауна? Соответствуют ли ключи ожидаемым?
- **Следование таксономии:** Использованы ли теги строго из разрешенного списка (или близкие к ним)?
- **Качество контента (Cringe-фактор):** Звучат ли айсбрейкеры естественно, неформально и уместно для свидания? Нет ли в них энциклопедичной "душноты"?

## 3. Выполнение тестов (Execution)
Прогоните тестируемый промпт через все примеры из тестовой выборки. Сделайте это последовательно. Зафиксируйте ответы LLM.

## 4. Анализ и Отчет (Analysis & Report)
Составьте отчет по результатам прогона:
1. Покажите краткую сводку по каждому примеру (Вход -> Выход).
2. Оцените каждый результат по выбранным критериям.
3. Выделите общие ошибки или галлюцинации (например, "LLM постоянно забывает закрывать скобки в JSON" или "Айсбрейкеры получаются слишком длинными").

## 5. Предложение улучшений (Optimization)
На основе выявленных проблем, предложите конкретные изменения в текст системного промпта:
- Добавьте более строгие ограничения (например, "Максимум 15 слов на айсбрейкер").
- Измените tone of voice, если результаты звучат неестественно.
- Добавьте few-shot примеры прямо в промпт, если LLM плохо справляется с форматом.

После согласования изменений с пользователем, обновите целевой файл (например, `LLM_STRATEGY.md`).
