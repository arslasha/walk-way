# Walk-Way Setup Guide

Это руководство описывает шаги, необходимые для развертывания проекта на новом устройстве (например, при переходе с Windows на Mac или наоборот).

## Предварительные требования
1. Установленный **Git**
2. Установленный **Docker** и **Docker Compose**
3. Настроенный агент **Antigravity** (со всеми необходимыми MCP-серверами: GitHub, FileSystem и др.)

## Шаги по установке

### 1. Клонирование репозитория
Склонируйте проект и перейдите в его директорию:
```bash
git clone https://github.com/arslasha/walk-way.git
cd walk-way
```

### 2. Настройка переменных окружения
Скопируйте пример файла конфигурации и при необходимости отредактируйте его:
```bash
cp .env.example .env
```
Убедитесь, что в `.env` прописаны корректные доступы к базе данных (настройки по умолчанию из `docker-compose.yml` уже прописаны в `.env.example`).

### 3. Запуск Docker-контейнеров
Соберите и запустите сервисы `db` (PostgreSQL с PostGIS) и `web` (Django):
```bash
docker-compose up -d --build
```

### 4. Применение миграций базы данных
После успешного запуска контейнеров, выполните миграции, чтобы создать все необходимые таблицы (включая таблицы приложения `places`):
```bash
docker-compose exec web python manage.py migrate
```

### 5. Создание суперпользователя (опционально)
Для доступа к админ-панели создайте суперпользователя:
```bash
docker-compose exec web python manage.py createsuperuser
```

### 6. Проверка работоспособности
Убедитесь, что сервер запущен, перейдя по адресу:
[http://localhost:8000/admin](http://localhost:8000/admin)

---
> **Важно:** При добавлении новых сервисов, переменных окружения или системных зависимостей, **не забывайте обновлять этот файл**, чтобы процесс развертывания оставался актуальным.
