# Task 017: Personal Folders & Place Collections (Backend & Frontend)

**Assignee:** Backend Agent & Frontend Agent (Coordinated)
**Status:** To Do

## Context
Вместо обычного плоского списка "Избранное" пользователи хотят структурировать свои любимые места по персональным тематическим папкам (коллекциям) (например, "Места для свиданий", "Где поработать с ноутбуком"). Место может находиться в нескольких коллекциях одновременно. Коллекция может быть публичной или приватной.
- **References:** `PLAN.md` (Phase 3), `docs/prd/phase3-social-accounts.md`.

## Requirements

### 1. Backend: Модель данных и API коллекции (`/api/v1/collections/`)
- **Модель `Collection`:**
  - `name` (CharField, до 100 символов, обязательно).
  - `description` (TextField, до 300 символов, опционально).
  - `owner` (ForeignKey на User, каскадное удаление).
  - `is_public` (BooleanField, по умолчанию `True`) — управляет доступностью папки для друзей.
  - `places` (ManyToManyField на `Place`, связанные места).
  - `created_at` (DateTimeField, авто-дата создания).
- **Эндпоинты API:**
  - `GET /api/v1/collections/` — возвращает список папок текущего пользователя (с аннотацией `places_count` — количеством мест в каждой папке). Поддерживает фильтрацию по владельцу.
  - `POST /api/v1/collections/` — создание папки (`name`, `description`, `is_public`).
  - `DELETE /api/v1/collections/{id}/` — удаление папки (только владельцем!).
  - `POST /api/v1/collections/{id}/add-place/` — добавление места (`place_id`) в указанную коллекцию.
  - `POST /api/v1/collections/{id}/remove-place/` — удаление места (`place_id`) из указанной коллекции.
- **Безопасность:** Доступ к CRUD папок имеют только их владельцы. Друзья могут совершать только `GET` запросы к публичным папкам другого пользователя.

### 2. Frontend: Менеджер папок в UI карточек
- **Кнопка "Сохранить" в карточке места:**
  - На каждой карточке места в `/explore` или в деталях места на `/map` заменить стандартное сердечко на кнопку "Сохранить" (иконка закладки/папки).
  - При клике на кнопку открывается всплывающее меню (Dropdown/Popover) со списком существующих папок пользователя.
  - У каждой папки в списке отображается чекбокс. Установка/снятие чекбокса мгновенно отправляет фоновый запрос (`add-place` / `remove-place`) к бэкенду и обновляет состояние в UI.
  - Внизу списка папок добавить кнопку "+ Создать новую папку", которая открывает инпут ввода названия внутри того же меню. При создании папка мгновенно появляется в списке с отмеченным чекбоксом для текущего места.

### 3. Frontend: Вкладка "Мои папки" в Профиле
- Во вкладке "Мои папки" личного кабинета `/profile` рендерить коллекции в виде элегантных карточек-превью (Grid Layout).
- Каждая карточка папки должна содержать:
  - Иконку папки и название.
  - Метку видимости: "Публичная" (зеленый бейдж) или "Приватная" (серый бейдж/иконка замка).
  - Количество мест.
  - Быстрый слайдер или коллапс-список сохраненных мест с возможностью удалить место из папки прямо отсюда.
  - Кнопку удаления самой папки с подтверждением ("Вы уверены?").

## Edge Cases & Error Handling
- **Попытка добавить дубликат:** Если бэкенд получает запрос на добавление места, которое уже есть в папке, запрос должен завершаться успешно (`200 OK`) без дублирования записи (использовать Django метод `.add()`).
- **Удаление мест:** При удалении места из базы данных (`Place`) оно должно автоматически исчезать из всех связанных коллекций (каскадная очистка связи M2M, стандартное поведение Django).
- **Пустые папки:** Красиво рендерить пустые папки в профиле с кнопкой быстрого перехода в `/explore` для поиска мест.
