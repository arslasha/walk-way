# 🚶‍♂️ Walk-Way

**Walk-Way** — это умный планировщик городских прогулок и свиданий, сочетающий в себе функции интеллектуального гео-навигатора и рекомендательного сервиса. 

В отличие от традиционных навигаторов, которые строят маршруты исключительно на основе физического расстояния, Walk-Way подбирает интересные места (кафе, парки, музеи, тайные споты) и выстраивает пешеходные маршруты вокруг **эмоционального контекста («вайба»)**, доступного времени и желаемого настроения пользователя.

---

## ✨ Основные особенности
- 🌈 **Вайб-ориентированный поиск:** Поиск мест по эмоциональным тегам (например: *ламповый, индустриальный, меланхоличный, футуристичный*), обогащенным с помощью LLM-анализа.
- 🗺️ **Интерактивная векторная карта:** Высокопроизводительная кастомная карта на движке `MapLibre GL JS`, поддерживающая темную и светлую темы в соответствии с визуальным стилем проекта.
- 🧳 **Умный конструктор маршрутов:** Удобная шторка с поддержкой drag-and-drop для выстраивания и сортировки промежуточных точек без необходимости постоянного переключения экранов.
- 📍 **Подборки «По пути»:** Алгоритм на базе PostGIS (`ST_DWithin`), который динамически анализирует построенную линию маршрута и предлагает добавить интересные места, находящиеся в пределах 150 метров от пути следования.

## 🛠️ Технологический стек

### Frontend
- **Ядро:** [Next.js 14/15](https://nextjs.org/) (App Router, React Server Components)
- **Стилизация:** Tailwind CSS
- **Управление состоянием:** [Zustand](https://github.com/pmndrs/zustand) (с персистентным хранением маршрута в `localStorage`)
- **Интерактивные карты:** [MapLibre GL JS](https://maplibre.org/) и [React Map GL](https://visgl.github.io/react-map-gl/)
- **Компоненты & Анимации:** Radix UI, Framer Motion, [Vaul](https://github.com/emilkowalski/vaul) (премиум-шторки)

### Backend & Data Engine
- **Ядро:** [Django 5.0](https://www.djangoproject.com/) / [Django REST Framework](https://www.django-rest-framework.org/)
- **База данных:** PostgreSQL с расширением **PostGIS** для геопространственных запросов (`djangorestframework-gis`)
- **Сбор данных:** Скрипты интеграции с KudaGo API
- **Интеллектуальный анализ:** Модели OpenAI API для автоматической классификации «вайбов» мест
- **Тестирование:** PyTest Django

## 🚀 Инструкция по запуску (Быстрый старт)

Для запуска проекта на вашем локальном компьютере должны быть установлены:
- **Docker** и **Docker Compose**
- **Node.js** (версии 18 и выше)
- **NPM** или **Yarn**

### Шаг 1. Клонирование репозитория и настройка окружения
Склонируйте проект и перейдите в его корневую директорию:
```bash
git clone https://github.com/arslasha/walk-way.git
cd walk-way
```

Создайте локальный файл конфигурации `.env` на основе шаблона `.env.example`:
```bash
cp .env.example .env
```
*(При необходимости откройте `.env` и добавьте ключи `OPENAI_API_KEY` и `ORS_API_KEY` для работы интеграций с картами и ИИ-генератором вайбов).*

---

### Шаг 2. Запуск бэкенда через Docker
Запустите контейнеры базы данных PostgreSQL (PostGIS) и Django-сервера:
```bash
docker-compose up -d --build
```
*Это действие соберет образ бэкенда, установит зависимости из `requirements.txt` и запустит сервер бэкенда на порту `8000`.*

---

### Шаг 3. Миграции и первоначальное наполнение БД
Примените схемы данных к геопространственной базе данных:
```bash
docker-compose exec web python manage.py migrate
```

Запустите встроенный импорт мест и событий через API KudaGo:
```bash
docker-compose exec web python manage.py parse_kudago
```
*(Скрипт автоматически скачает и сохранит базовый набор категорий, тегов и мест с их точными координатами).*

*(Опционально)* Для разметки мест эмоциональными тегами с помощью ИИ запустите команду обогащения:
```bash
docker-compose exec web python manage.py generate_vibes
```

---

### Шаг 4. Запуск автоматических тестов
Чтобы убедиться, что бэкенд и пространственные фильтры работают корректно, запустите тесты:
```bash
docker-compose exec web pytest
```

---

### Шаг 5. Установка и запуск фронтенда
Откройте вторую вкладку терминала, перейдите в папку `frontend`, установите npm-пакеты и запустите Next.js в режиме разработки:
```bash
cd frontend
npm install
npm run dev
```

После этого откройте в браузере **[http://localhost:3000](http://localhost:3000)**. Приложение готово к работе! Светлая/темная тема будет автоматически синхронизироваться со стилем интерактивной карты.

## 🎨 Дизайн-концепция (Dramatic Urbanism)
Интерфейс Walk-Way спроектирован с упором на премиальный пользовательский опыт:
- **Широкие скругления** элементов (`rounded-[40px]`) для создания эффекта дружелюбного и «мягкого» интерфейса.
- **Глубокие тени** и контрастный темный режим для ночных прогулок.
- **Микроанимации** интерактивных кнопок и переходов.
- Подробные токены, шрифты и стили задокументированы в файле [DESIGN.md](file:///c:/GitHub-Repositories/walk-way/DESIGN.md).
