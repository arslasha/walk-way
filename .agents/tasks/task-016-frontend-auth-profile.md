# Task 016: Auth Forms, User Dashboard & Profile Editing UI (Frontend)

**Assignee:** Frontend Agent
**Status:** Completed

## Context
С внедрением JWT и профилей на бэкенде, фронтенду требуется реализовать удобные, красивые интерфейсы для входа, регистрации, дашборда профиля и настройки двухфакторной аутентификации (2FA). Весь интерфейс должен соответствовать концепции **Dramatic Urbanism**, описанной в `DESIGN.md`.
- **References:** `PLAN.md` (Phase 3), `docs/prd/phase3-social-accounts.md`, `DESIGN.md`.

## Requirements

### 1. Состояние авторизации (Zustand Auth Store)
Создать стор `useAuthStore` для управления глобальным состоянием сессии:
- Свойства: `user` (данные профиля), `accessToken`, `refreshToken`, `isAuthenticated`, `is2faRequired`, `preAuthToken` (сессионный токен 2FA).
- Действия: `login`, `register`, `logout`, `verify2FA`, `updateProfileState`, `refreshSession`.
- Настроить автоматический перехват 401 ошибок во всех fetch-запросах для прозрачного обновления access-токена с помощью refresh-токена. В случае истечения refresh-токена перенаправлять пользователя на страницу входа с выводом тоста: *"Ваша сессия истекла. Войдите снова"*.

### 2. Формы Входа и Регистрации (`/auth/login`, `/auth/register`)
- **Дизайн:** Использовать стеклянные карточки (glassmorphism) со скругленными углами, мягким размытием фона (`backdrop-blur`) и тонкими контрастными границами.
- **Интерактивность:** 
  - Валидация полей на лету (проверка почты, совпадение паролей, минимальная длина пароля).
  - Красивый спиннер загрузки при отправке формы.
  - Тосты для вывода ошибок авторизации (например, *"Неверный пароль"*, *"Пользователь с таким email уже существует"*).
- **Экран ввода OTP (2FA):** Если бэкенд возвращает статус `2fa_required`, форма логина должна плавно (через CSS transition/Framer Motion) переключаться на экран ввода 6-значного одноразового кода.

### 3. Личный Кабинет / Дашборд (`/profile`)
- **Заголовок профиля (Header):**
  - Аватар пользователя (круглый, с эффектом легкого свечения при наведении, при отсутствии аватара — сгенерированная иконка с инициалами).
  - Никнейм (`@nickname`), имя и короткое описание (био).
  - Вкладки (Tabs): "Мои прогулки", "Мои папки", "Настройки безопасности".
- **Модальное окно редактирования профиля:**
  - Форма смены никнейма и био.
  - Интерактивная зона загрузки аватара (Drag & Drop) с возможностью предпросмотра изображения и кропом перед отправкой (использовать легкую библиотеку типа `react-image-crop` или HTML5 Canvas).
- **Вкладка безопасности (2FA Setup):**
  - Кнопка "Включить 2FA". При нажатии открывается пошаговый диалог:
    1. Показ QR-кода (сгенерированного на основе `otp_uri` с бэкенда) и текстового ключа.
    2. Поле ввода 6-значного TOTP-кода для подтверждения.
    3. Вывод 5 кодов восстановления (с кнопкой "Скопировать") и предупреждением о важности их сохранения.
  - Если 2FA уже включена — показывать статус "Защищено" и кнопку "Отключить 2FA" с подтверждением через пароль.

## Edge Cases & Error Handling
- **Сессионные сбои:** При потере сети во время отправки аватара выводить Toast: *"Не удалось загрузить изображение. Проверьте интернет-соединение"*.
- **Предотвращение дублирования кликов:** Блокировать кнопки отправки форм (`disabled={isLoading}`) для исключения повторных запросов.
- **Адаптивность:** Убедиться, что дашборд профиля и формы выглядят безупречно как на ультрашироких экранах, так и на мобильных устройствах.
